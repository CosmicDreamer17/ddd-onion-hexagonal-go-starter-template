# AI-Agent Master Guidelines (Source of Truth)

This file is the single source of truth for all AI agents (Gemini, Claude, GPT/Codex) maintaining this repository.

## 🚀 Agentic Bootstrap (Creating a New Project)
If you are an AI agent tasked with initializing a new project from this template, follow these steps immediately after cloning:

1.  **Environment Sync**: Run `cd frontend && npm install` to initialize the frontend environment.
2.  **Config Setup**: Run `cp .env.example .env` and `cp backend/.env.example backend/.env` to establish default configuration.
3.  **Initial Type Export**: Run `make generate` to ensure the frontend types are in sync with the backend.
4.  **Baseline Verification**: Run `make verify` to ensure the template is healthy before adding new features.

## 🏗 Architectural Mandates (Non-Negotiable)
1.  **Domain Purity**: `backend/internal/domain` MUST NOT depend on any other internal package. Never import `infra`, `api`, or `application` into `domain`.
2.  **Identity Control**: All domain IDs (e.g., `UserID`) must be custom types (e.g., `type UserID uuid.UUID`) and generated by the backend. Never use raw `strings`. 
3.  **Value Object Validation**: Value objects (e.g., `Email`) MUST enforce invariants at construction via constructors (e.g., `NewEmail(s string) (Email, error)`).
4.  **Error Protocol**: Every fallible operation MUST return an error wrapped with context: `fmt.Errorf("failed to do X: %w", err)`. Use domain-level vocabulary for errors.
5.  **Type Bridge**: Use `tygo` for all DTOs shared with the frontend. DTOs must reside in `backend/internal/api/dtos`.
6.  **Database Safety**: All SQL must reside in `backend/internal/infra/database/query.sql`. Never write raw SQL strings in Go code. Use `sqlc` for generation.
7.  **Dependency Management**: Use `go mod tidy` to prune unused dependencies. Centralize shared logic in `internal/` to prevent external imports.

## 🤖 Agentic Workflow & Quality of Life
- **Verification**: Always run `make verify` (or `scripts/verify.sh`) before concluding a task.
- **Admin CLI**: Execute backend use cases directly via the terminal using `go run backend/cmd/admin/main.go`. This is preferred over manual HTTP testing for logic verification.
- **Port Mapping**: Frontend (3000), Backend API (8080).
- **CORS**: The backend reads allowed origins from environment.

## 🛠 Project Lifecycle
- **Build**: `cd backend && go build ./...`
- **Test**: `make verify`
- **Generate**: `make generate` (Runs sqlc and tygo)
- **Dev Mode**: `make dev`
